Failover Flow — Windows Down → GEEKOM Takes Over
Step 0 — Detection
🚨 Consul detects Windows PC is DOWN
Consul health checks fail for yamil-frontend on the primary node (windows-pc). After 30s, service goes critical. Consul watch handler triggers on QNAP. ntfy sends urgent push notification.
consul watch: yamil-frontend → critical → failover-handler.sh
Step 1 — Verify Standby
✅ Consul picks failover target
Failover handler queries Consul health API. Prefers GEEKOM (amd64, same arch). If GEEKOM node is also unhealthy, falls through to Mac Mini. Records target in KV store.
curl consul:8500/v1/health/node/geekom → passing → TARGET=geekom
Step 2 — Sync Code
📥 Pull latest code on GEEKOM
Pull latest commits from all 3 project repos. Images were pre-built, so no rebuild needed (same amd64 architecture).
ssh geekom "cd ~/project/parser_lite && git pull"
Step 3 — Restore Secrets
🔑 Copy latest secrets from QNAP
SCP the .env, .env.secrets, and .env.prod files from QNAP NAS dev-transfer share to GEEKOM. These are synced hourly from the active host.
scp admin@192.168.0.102:/share/dev-transfer/dot-env* ~/project/...
Step 4 — Restore Database
🗄️ Restore latest pg_dump from QNAP
The QNAP db-backup container creates dumps every 6 hours. Copy the most recent dump and restore into GEEKOM's Postgres. Maximum data loss: 6 hours.
pg_restore -U postgres -d message_weaver --clean --if-exists < /tmp/latest.dump
Step 5 — Start Services
🚀 Start all Docker Compose stacks
Start Y.A.M.I.L first (Vault + Postgres must initialize), then DriveSentinel, then Memobytes. Pre-built images mean no build step — just docker compose up.
docker compose up -d (all 3 projects)
Step 6 — Switch Context
🔀 Switch MacBook Docker context to GEEKOM
The MacBook Air's Docker CLI now points to GEEKOM. All docker commands from Claude Code go to the new active host.
docker context use geekom
Step 7 — Update Monitoring
📊 Redirect QNAP backups to GEEKOM
Update the ACTIVE_HOST variable in QNAP's db-backup container so future pg_dumps target GEEKOM (192.168.0.113) instead of Windows (192.168.0.101).
ACTIVE_HOST=192.168.0.113 → docker compose restart db-backup
Step 8 — Verify
✅ Confirm all services are healthy
Run health checks on all endpoints. Verify Grafana shows GEEKOM containers as UP. Total failover time: ~3 minutes.
curl http://192.168.0.113:9080 && echo "Failover complete!"
QNAP Monitoring Stack (192.168.0.102)
📊
Grafana
:3000
Dashboards and alerting. Connects to Prometheus (metrics) and Loki (logs). Displays service health, resource usage, API latency, database stats, and HA status across all Docker hosts. Fires alerts via ntfy and email.
📈
Prometheus
:9090
Metrics collection engine. Scrapes 34 targets every 15-30 seconds. Stores 90 days of time-series data. Targets include Docker daemon metrics (:9323), node/Windows exporters (:9100/:9182), blackbox HTTP/TCP probes, and 5 alert rules via Alertmanager.
📝
Loki
:3100
Log aggregation. Receives container logs from Promtail agents on each Docker host. Queryable through Grafana Explore. Persists logs even after container restarts.
🔍
Blackbox Exporter
:9115
External uptime probes. HTTP probes test service endpoints, TCP probes test Postgres and Redis ports. Provides an outside perspective — if QNAP can't reach a service, it's truly down.
🔔
ntfy
:8090
Self-hosted push notification server. Grafana alerts POST to ntfy, which pushes notifications to your phone. Subscribe to the yamil-alerts topic in the ntfy app.
🏛️
Consul Server
:8500
HA orchestration. Server on QNAP with clients on each machine. Health checks every 10-15s on all services. Watches trigger failover scripts. KV store tracks active node. DNS on :8600 resolves *.service.consul to healthy nodes.
🔄
Git Mirror
cron / 1h
Bare mirror clones of all 5 GitHub repos (parser_lite, FlashCards, ai-bot, Ai-Tools, yamil-browser). Fetches every hour using GitHub PAT. Ensures code is available even if GitHub is unreachable.
👀
Watchtower
daily 4am
Auto-updates all container images daily at 4am. Pulls latest tags, recreates containers, removes old images. Sends notifications to ntfy when updates are applied.
☁️
Cloudflared
tunnel
Cloudflare tunnel for external access. Routes grafana.yamil-ai.com to Grafana and ntfy.yamil-ai.com to ntfy. Uses Docker service names for routing.
✅
Uptime Kuma
:3001
Lightweight uptime monitoring with web dashboard. Alternative view alongside Blackbox/Prometheus. Monitors internal and external endpoints with status pages.
🗄️
DB Backup
cron / 6h
Postgres container that runs pg_dump against the active Docker host every 6 hours. Stores dumps on NAS. Retains 7 daily + 4 weekly + 3 monthly. Alerts if backup is older than 12 hours.
