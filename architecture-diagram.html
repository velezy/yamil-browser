Infrastructure Components
💾
PostgreSQL 17 + pgvector
:5432 (local) / :5433 (RDS tunnel)

Primary relational database with vector search extension for hybrid storage.

Key Tables
  • users — Accounts, roles, preferences
  • flashcards — Card front/back, difficulty, schedule
  • collections — Flashcard decks and organization
  • quiz_sessions — Quiz attempts, scores, answers
  • study_sessions — Study progress and scheduling
  • conversations — AI chat history
  • documents — Uploaded file metadata
⚡
Redis 7
:6379

In-memory data store powering background jobs, caching, and real-time features.

  • Celery task broker and result backend
  • Job progress pub/sub channels
  • Session and response caching
  • Rate limiting counters
  • Real-time progress updates via Redis channels
🧠
Ollama
:11434

Local LLM inference server with multi-model support and OpenAI-compatible API.

  • llama3.2:3b — Fast tier (quick responses)
  • llama3.1:8b — Standard tier (general use)
  • gemma3:4b — Alternative model
  • Parallel processing (OLLAMA_NUM_PARALLEL=4)
  • GPU acceleration (Metal on macOS)
📦
Celery + Beat
2 workers · Redis broker

Distributed task queue for background AI generation and scheduled jobs.

  • Study guide generation (rate: 10/min)
  • Quiz generation (rate: 20/min)
  • Web crawling and PDF import
  • Beat scheduler: jobs every 60 seconds
  • 3 priority queues: high, default, low
☁️
AWS RDS
SSH tunnel via EC2

Production PostgreSQL database on AWS, accessed via SSH tunnel.

  • RDS host: memobyte-dev-postgres.*.rds.amazonaws.com
  • EC2 bastion: SSH key-based tunnel
  • Local port forwarding: 5433 → RDS 5432
  • All production user data and study progress
🛡️
Apache APISIX (Production)
:9080 · optional profile

Production API gateway with etcd configuration store.

  • Request rate limiting
  • JWT authentication validation
  • Prometheus metrics (:9091)
  • Automatic route initialization
  • Enabled via --profile gateway
